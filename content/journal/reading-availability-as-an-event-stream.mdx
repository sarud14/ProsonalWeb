## Optimism has a price

Treating availability as client state made the UI feel instant — and occasionally oversold inventory under load. The server was the only honest source of truth.

## Server-driven, event-shaped

Modelling availability as an event stream let the server stay authoritative while the client subscribed to changes. Fewer states to reason about, and no double-booking class of bug.
